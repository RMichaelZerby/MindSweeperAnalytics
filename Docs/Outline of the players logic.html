Outline of the Players Logic


Outline of the play


1 Initialize the game interface

2 Initialize the game configuration

3 Begin the game

4 Initial guess of any cell

5 While we are still playing the game

5.1 Read the array of cells from the game interface

5.2 For all the cells in the array read

5.3 Update the corresponding shadow cell's display

5.3.1 If the cell has changed its display state

5.3.1.1 For all the adjacent cells

5.3.1.1.1 Subtract one from number of adjacent cells that are covered

5.3.2 If the cell is in play and is uncovered

5.3.2.1 Add this cell to the list of uncovered cells

5.4 If there are any uncovered cells found

5.4.1 For all the uncovered cells discovered

5.4.1.1 "Find New Bombs"

5.4.2 If we found all the bombs

5.4.2.1 For all the remaining covered cells that are not bombs

5.4.2.1.1 Select this cell through the game interface

5.4.2.2 Then I've won (go to step 5.7 )

5.4.3 If we have found any new bombs

5.4.3.1 "Find New Safe Cells"

5.5 If there were any safe cells found

5.5.1 For all the safe cells found

5.5.1.1 Select the cell, through the game interface

5.6 If there were no safe cell found

5.6.1 "Make A Guess"

5.6.2 Select the guessed cell(s), through the game interface

5.7 If I am not still playing(won or lost)

5.7.1 Write out the results of the game.


Outline of "Find New Bombs"


1 Count the number of adjacent covered cells to this one

2 If the count of covered cells is equal to the cell's value

2.1 For all the adjacent covered cell

2.1.1 Add this cell to the list of bombs

2.1.2 Mark this cell as a bomb

2.1.3 For all adjacent cells of the bomb

3 Increment the adjacent bomb count


Outline of "Find New Safe Cells"


1 For all the bombs I have found

2 If the bomb is still in play (not neutralized)

2.1 For all the cells adjacent to the bomb

2.2 If the cell is in play (uncovered and not neutralized)

2.2.1 If the cell's value is reduced to zero

2.2.1.1 Mark this cell as not in play (or neutralized)

2.2.1.2 For all the cells adjacent to this one.

2.2.1.2.1 If the cell is covered and not a bomb

2.2.1.2.1.1 Add it to the list of safe cells to uncover

2.2.1.3 If all the adjacent cells to the bomb are no longer in play

2.2.1.3.1 Mark this bomb as not in play (or neutralized)

3 Return the new list of safe cells.

Outline of "Make A Guess"


1 For all the cells

1.1 If the cell is still covered and not a bomb.

1.1.1 For all adjacent cells

1.1.1.1 If the cell has a value (poses a threat)

1.1.1.1.1 Lowest value of bomb threat

1.1.2 If this cell has a direct bomb threat

1.1.2.1 Collect the cell with the lowest threat to the sorted list of direct threats

1.1.3 If this cell does not have a direct bomb threat

1.1.3.1 Collect this cell to the list of cells with an indirect threat

2 "Any Sure Things"

3 If we did not find any sure things

3.1 Calculate the minimum bomb threat for indirect cells

3.2 For all the cells in the indirect threat

3.2.1 Add or replace this cell to the sorted list of key as minimum threat

3.3 For all the cells in the ascending sorted list

3.3.1 If the value of this cell is the same as the first cell in the list.

3.3.1.1 Increment the count of cells to choose from.

3.3.2 Else

3.3.2.1 Stop the search

3.4 Randomly pick a cell from the cells to choose from

3.5 Return the cell we have guessed has the lowest threat

4 Return list of sure things


Outline of "Any Sure Things"


1 For all the input with a direct threat aligned horizontally

1.1 Collect consecutive covered cells as series of inputs

1.2 Convert the consecutive input into coupled gates

1.3 "Composite Input" - using coupled gates

2 If we have not discovered all the bombs

2.1 For all the input with a direct threat aligned vertically

2.1.1 Collect consecutive covered cell as series of inputs

2.1.2 Convert the consecutive input into coupled gates

2.1.3 "Composite Input" - using coupled gates

3 If we have discovered all the bombs

3.1 return the list of all the cells that are not bombs (Then I've won)

4 Return list of solved inputs


Outline of "Composite Input"


1 For the set consecutive covered cells

1.1 Create a keyed gate of type OR; AND or NONE

1.2 If the gate is of type OR or AND

1.2.1 For all the adjacent cells that are covered

1.2.1.1 Create a keyed input orientation opposite to the gates

1.2.1.2 Add the inputs to the gate

1.2.1.3 If the gate expresses a unique set of inputs

1.2.1.3.1 Add this gate to the set of gates

1.2.1.3.2 Add any new inputs to the set of inputs

2 For the set of inputs that share two or more gates

2.1 Mark the input as tightly coupled

3 For the set of gates

3.1 Create a key for this gate based its inputs that are tightly coupled

3.2 If this gate has a key comprised of more then one input

3.2.1 Add this gate to the list of gates to solve for

4 For all the gates to be solved for

4.1 Deduce inputs states

5 Return the list of solved inputs
